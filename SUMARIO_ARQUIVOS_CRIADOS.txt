📋 PROBLEMAS IDENTIFICADOS E SOLUÇÕES
================================================================================

PROBLEMA #1: QR Code não atualiza em tempo real
┌─────────────────────────────────────────────┐
│ Antes: GET /whatsapp/qr (retorna imagem)    │
│ Depois: POST /whatsapp/connect + polling    │
│         └─ Atualiza a cada 5 segundos       │
└─────────────────────────────────────────────┘

PROBLEMA #2: Sessão WuzAPI não inicia automaticamente
┌─────────────────────────────────────────────┐
│ Antes: Usuário clica botão, nada acontece   │
│ Depois: Backend chama startSession()        │
│         └─ Inicia sessão automaticamente    │
└─────────────────────────────────────────────┘

PROBLEMA #3: Agente não responde mensagens
┌─────────────────────────────────────────────┐
│ Antes: Webhook pode estar desregistrado     │
│ Depois: Script de diagnóstico mostra o quê  │
│         └─ Identifica problema exatamente   │
└─────────────────────────────────────────────┘

🆘 TROUBLESHOOTING RÁPIDO
================================================================================

Problema: "QR Code não aparece"
Solução:
  1. Ver console do navegador (F12)
  2. Verificar se POST /whatsapp/{id}/connect retorna erro
  3. Verificar se WuzAPI está rodando: curl http://localhost:21465/health
  4. Ver logs: tail -f storage/logs/laravel.log

Problema: "Agente não responde"
Solução:
  1. Rodar: php artisan diagnose:whatsapp-agent --integration-id=1
  2. Script mostra exatamente o que está faltando
  3. Comum: AI não ativada, API key inválida, webhook errado

Problema: "Erro ao conectar"
Solução:
  1. Ver logs: grep -i whatsapp storage/logs/laravel.log
  2. Verificar WuzAPI URL está correta em .env
  3. Verificar token do WuzAPI
  4. Reiniciar WuzAPI container
